# Research and Recommend

Use this flow when the user wants analysis, options, or a recommendation.

1. gather only the information needed
2. separate facts from assumptions
3. present 2-3 options when real tradeoffs exist
4. give a recommendation with reasoning
5. do not execute unless the user explicitly asks

## Output shape

- findings
- options
- recommendation
- risks / unknowns

## Scope boundary

If the user approves execution, switch to [execute-approved-task.md](./execute-approved-task.md).
