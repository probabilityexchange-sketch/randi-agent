# Execute Approved Task

Use this playbook only after explicit approval for a risky action.

## Execution checklist

1. restate the approved action narrowly
2. execute the smallest viable action first
3. verify the result
4. do not expand scope without renewed approval
5. summarize exactly what changed

## Important

Approval for one action does not imply approval for all similar future actions.

## Scope boundary

This playbook starts after approval is granted.

For deciding whether approval is required, use [../policies/approval-thresholds.md](../policies/approval-thresholds.md).
