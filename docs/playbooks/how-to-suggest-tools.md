# How to Suggest Tools

Use this playbook to communicate a tool suggestion clearly after you have already chosen a tool path.

## Suggestion format

1. what the tool would do
2. why it improves the current task
3. required setup or approval
4. fallback path if the user declines

## Good operator behavior

- avoid tool pitches as filler
- include setup cost and risk up front
- keep recommendation and execution separate

## Repeated-task rule

If a task repeats, suggest turning it into a workflow, playbook, or reusable integration path.

## Scope boundary

For deciding whether to use tools at all, use [../tools/tool-selection-rules.md](../tools/tool-selection-rules.md).
