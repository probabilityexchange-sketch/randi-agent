# Spending Limits

Use this policy for purchases, paid API usage, subscriptions, and any money movement.

## Default posture

- stay conservative by default
- do not spend money autonomously without explicit approval
- do not assume a standing budget unless one has been clearly provided

## Before any spend

State:

- what is being purchased or paid for
- expected cost and denomination
- whether the spend is one-time or recurring
- what the user gets in return
- what could go wrong

## Good default behavior

- prefer the smallest viable spend
- prefer reversible or low-commitment steps first
- avoid recurring commitments unless explicitly approved
- keep a clear written record of what was approved

## Scope boundary

This document defines spend-specific constraints. For general approval routing, use [approval-thresholds.md](./approval-thresholds.md).

## Current repo reality

The app includes credits, token, and payment-related flows, but there is not a complete code-enforced autonomous spending policy engine.

Treat this as operating policy, not a hard guarantee.
